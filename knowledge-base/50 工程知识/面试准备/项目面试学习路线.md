# 项目面试学习路线

## 一句话项目定位

WebAgent Clean 是一个多用户 Agent 工作台：用户通过网页会话发起任务，系统用 PlanAgent 规划、ReactAgent 调工具执行，结合 RAG、Skill、MCP 动态工具、AIO Sandbox、浏览器视觉观察和文件预览，完成从思考到产物交付的闭环。

## 简历可以写的方向

- 设计并实现多用户 Agent 编排链路，拆分上下文准备、规划、ReAct 执行、Hook 装配和工具运行态清理，支持同步与 SSE 流式对话。
- 引入运行时 TodoState 和最终回答门禁，通过 `todo_write` 记录任务状态、验收信号和证据，减少多步任务提前结束。
- 接入 AIO Sandbox，构建用户级沙箱复用、文件同步、Office 预览和同源沙箱视图代理，支持浏览器、终端、VSCode、文件树等工作台能力。
- 实现工具系统扩展：本地工具、Skill 渐进式披露、MCP 动态工具、子代理和视觉观察统一接入 ReactAgent。
- 建设 RAG 知识库链路，支持文档上传、解析、切片、Embedding、可选 Milvus、Query Rewrite 和 BGE Rerank。
- 优化前端 Chat 工作台，支持 SSE 工具行动流、右侧工具 dock、文件产物图库、图片 lightbox、Office/PDF/文本统一预览。

## 必学 1：整体架构

先能画出这条链路：

```text
Vue Chat → AgentController → AgentServiceImpl
→ AgentTurnContextService → AgentPlannerService
→ ReactAgentFactory / ReactAgentHookService
→ ReactAgent → Tool / MCP / Sandbox / RAG
→ SSE 事件和 assistant 消息保存
```

要会解释为什么拆分：`AgentServiceImpl` 如果同时管工具、Skill、MCP、Hook、模型调用，会很难维护；拆成服务后，同步和流式路径复用同一套上下文与执行器工厂。

对应阅读：[[Agent 编排模块]]、[[计划与 ReAct 执行]]、[[创建会话并完成一次对话]]。

## 必学 2：Agent 执行闭环

重点理解四层：

- PlanAgent：任务前规划，给目标和策略。
- ReactAgent：执行工具循环，拿 observation 决策下一步。
- Hook：处理边界事件，例如图片观察、最终回答门禁（TodoState + 证据自检）。
- TodoState：执行中状态账本，记录 evidence，防止没做完就答复。

面试回答可以说：我没有把 Todo 做成调度器，而是作为“可验证任务状态”，调度仍由 ReactAgent 和工具循环负责，避免职责混乱。

对应阅读：[[计划与 ReAct 执行]]、[[Agent 工具调用]]、[[工具系统模块]]。

## 必学 3：Sandbox 和文件系统

要能讲清楚：

- 为什么从会话级 Sandbox 转向用户级 Sandbox。
- `UserSandboxEntity` 如何保存 sandboxId 和 endpoint。
- 为什么前端不能直接访问 AIO endpoint。
- `/sandbox-view/{token}/...` 如何代理 HTTP 和 WebSocket。
- 用户上传目录和 Sandbox `/home/gem` 如何同步。

面试高频追问：如果沙箱重建 endpoint 变了怎么办？答案是 token 只保存 userId，代理每次按用户查最新 endpoint。

对应阅读：[[Sandbox 模块]]、[[用户 Sandbox 创建与恢复]]、[[文件与工作区模块]]。

## 必学 4：工具系统和 MCP

要能讲清楚 Tool 接口：工具定义给模型，执行返回字符串 observation。MCP 工具不是特殊通道，而是包装成 `RealMcpTool` 后进入同一工具系统。

还要能讲安全边界：用户动态 MCP 默认要求 HTTPS、禁止私网，避免 SSRF；shell transport 只在 Sandbox 内运行。

对应阅读：[[工具系统模块]]、[[工具目录]]、[[Skill 与知识检索工具]]。

## 必学 5：RAG

要能讲上传和检索两条链：

- 上传：文件保存 → 解析 → 切片 → Embedding → Milvus/Noop → 元数据保存。
- 检索：Query Rewrite → 多 query Embedding → 向量召回 → 去重 → Rerank → 注入上下文。

要主动说明当前默认 `RAG_MILVUS_ENABLED=false`，适合本地启动，不代表完整 RAG 演示状态。

对应阅读：[[RAG 知识库模块]]、[[文档摄取与切片]]、[[查询改写与重排]]。

## 必学 6：前端实时体验

要能讲：

- 为什么不用 EventSource，而用 `fetch + ReadableStream`。
- SSE 事件如何展示 plan、thinking、tool_call、observation、answer。
- 右侧工具 dock 如何承载沙箱和工作区。
- 文件产物如何从工具事件中提取并去重展示。
- `deliver_to_user` 如何防止过程截图污染最终结果。

对应阅读：[[前端模块]]、[[Office 文件预览]]、[[文件上传与预览]]。

## 高频面试问答

**Q：这个项目最难的点是什么？**

A：难点不是单次调用模型，而是让模型、工具、沙箱、文件、RAG 和前端实时状态形成闭环。比如模型调用工具后返回 observation，Hook 可能注入图片观察，TodoState 可能阻止提前最终回答，前端还要把这些事件恢复成可读的行动流。

**Q：怎么保证多用户隔离？**

A：业务数据都带 userId；会话、知识库、上传文件、Sandbox 和 MCP 配置都要按当前用户校验。沙箱从会话级改成用户级后，endpoint 通过 `UserSandboxEntity` 和用户级 key 管理，前端只拿短期 token 访问同源代理。

**Q：为什么要 Hook？**

A：Hook 把工具执行前、执行后、结束前这些横切逻辑抽出来。比如图片观察不应该塞进工具本身，也不应该每轮扫描消息；用 PostToolUseHook 只在 `view_image` 后触发更清晰。

**Q：RAG 怎么做的？**

A：文档上传后解析切片、Embedding、向量写入；查询时先 rewrite，多 query 向量召回后去重，再 BGE rerank。Milvus 是可选依赖，关闭时系统可启动但检索为空。

**Q：你会怎么继续优化？**

A：补端到端测试，尤其是 SSE、真实 AIO 沙箱视图、MCP 工具生命周期和多用户越权；再考虑 TodoState 持久化、普通多 tool call 支持和移动端工具 dock 体验。
