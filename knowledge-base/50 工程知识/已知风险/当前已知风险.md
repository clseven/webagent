# 当前已知风险

## 未提交代码风险

当前工作区存在大量未提交修改和未跟踪文件。知识库已按当前文件内容更新，但这些事实在 git 提交前仍可能变化。写简历或面试前建议先确认最终代码状态。

## 沙箱视图代理风险

- HTTP 代理和 WebSocket 桥接已有路径/token 测试，但仍需要真实 AIO noVNC、code-server、terminal 端到端验证。
- token 只存在内存中，服务重启后旧 iframe URL 会失效，需要前端重新请求 view-url。
- 当前代理按用户 token 访问用户最新 endpoint，若 token 泄露，在有效期内可能访问该用户沙箱视图。
- Location 改写和 WebSocket 子协议透传需要继续覆盖更多 AIO 页面形态。

## 多租户和路径风险

- 文件、知识库、会话、应用、Sandbox 都必须按 userId 校验归属。
- 文件路径需要持续防止 `../`、绝对路径、URL 编码绕过。
- docId/sessionId/appId 等直接传参接口需要系统性越权测试。

## RAG 风险

- `RAG_MILVUS_ENABLED=false` 是当前默认值，开发环境能启动，但 RAG 检索效果会退化为空向量召回。
- Query Rewrite 和 Rerank 都依赖外部本地服务；服务不可用时需要确认退化行为和日志。
- 文档解析和 Office 转换对大文件、复杂格式仍可能耗时或失败。

## MCP 风险

- 用户动态 MCP 默认禁止 HTTP 和私网访问是必要的 SSRF 防线，不应为了方便随意打开。
- shell transport 在 Sandbox 中启动 supergateway，涉及端口分配、长进程清理和协议解析，真实稳定性需要压测。
- 新增 MCP server 后工具出现有时需要下一轮 Agent loop，产品提示需保持清楚。

## Agent 执行风险

- TodoState 是内存态，服务重启后不会保留。
- 普通工具调用当前仍以单 tool call 为主，并行主要依赖 `run_subagent`。
- 子代理后台执行需要避免最终回答早于后台结果消化。
- Hook 注入消息很强大，但新增 Hook 时要注意同步/流式两条路径一致。

## 前端风险

- Chat 页面和 `style.css` 变动很大，缺少浏览器自动化测试。
- 右侧 dock 在移动端隐藏，移动端沙箱/工作区体验需要单独验证。
- 图片 lightbox、Office 转换、工作区树和 SSE 事件恢复都依赖较多运行时状态。
