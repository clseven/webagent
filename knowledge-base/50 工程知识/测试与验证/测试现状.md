# 测试现状

## 已覆盖

- 文件上传先本地用户级目录持久化，再同步 Sandbox。
- 文件下载和删除按会话所属 userId 访问用户级 uploads 目录。
- 知识库和工作区 Office 预览类型映射。
- 前端截图产物过滤：`browser_screenshot` 只有 `deliver_to_user=true` 才展示。
- 图片图库最多先展示 4 张，其余折叠计数。
- 沙箱视图 token 签发和过期。
- 沙箱视图代理路径还原和 Location 改写。
- TodoState 状态校验、摘要生成、未完成 todo 防删除。
- `todo_write` 工具参数协议。
- `FinalTodoGuardHook` 最终回答门禁。
- ReactAgent Hook 服务注册 TodoState 门禁。
- ReactPromptAssembler 的浏览器截图、Skill、MCP、todo_write 等 section 约束。
- DeepSeek 错误码映射、重试、流式重试边界和异常结束原因。
- 用户工作区路径和文件名清理。
- 用户工作区同步、旧知识库文件迁移、Sandbox 创建时先迁移再恢复工作区。

## 主要缺口

- 完整同步和 SSE 对话仍缺少端到端测试，尤其是前端恢复历史事件、停止流式、后台子代理结果消化。
- 沙箱视图 HTTP 代理和 WebSocket 桥接缺少真实 AIO 端到端测试；当前主要覆盖路径和 token 规则。
- MCP 动态工具缺少覆盖“新增 server → 工具出现 → 调用 → 移除”的完整集成测试。
- RAG 检索在 Milvus 开启状态下的 query rewrite、向量召回、rerank、退化路径仍需更直接测试。
- 多用户越权测试矩阵还不完整，特别是 docId、sessionId、appId、sandbox view token 和文件路径。
- 前端没有完整浏览器自动化测试和构建期静态检查。

## 面试准备建议

面试时可以主动说明：这个项目已有较多单元测试覆盖关键规则，但端到端和真实 AIO/MCP 集成测试仍是下一阶段工程化重点。这样既展示了测试意识，也不会把当前系统包装成完全生产化。
