# 待确认问题

## 沙箱视图

- 真实 AIO 环境下 noVNC、terminal、code-server 是否都能通过 `/sandbox-view/{token}/...` 完整加载？
- WebSocket 桥接在长时间会话、断线重连和大输出时是否稳定？
- token TTL 30 分钟是否符合产品体验？是否需要刷新机制？
- 是否需要把沙箱视图 token 绑定更多信息，例如 user-agent、sessionId 或一次性 nonce？

## Agent 执行

- TodoState 是否需要跨请求或跨服务重启持久化？
- `run_subagent` 后台结果消化最多轮数、超时时间和前端展示是否需要产品化？
- 普通工具多 tool call 是否要正式支持？如果支持，需要重新设计执行器循环和事件协议。
- Hook 注册顺序是否需要显式优先级？

## MCP

- 用户动态 MCP 配置是否需要加密存储 API Key 和 headers？
- shell transport 的 supergateway 长进程如何做更强的生命周期回收？
- MCP 工具命名冲突和权限提示是否需要更可解释的前端展示？

## RAG

- Milvus 默认关闭是否只是本地开发临时配置？面试或演示前是否需要切换为启用并准备数据？
- Query Rewrite 和 Rerank 的服务不可用时，前端和 Agent 应展示什么降级信息？
- 文档切片大小 500/50 是否适合当前简历或项目文档类知识库？

## 前端

- Chat 页右侧 dock 在移动端隐藏后，用户如何访问沙箱和工作区？
- 文件产物图库是否需要支持更多排序、筛选或批量下载？
- 是否需要为 SSE 事件恢复、停止、重试增加 Playwright 测试？
