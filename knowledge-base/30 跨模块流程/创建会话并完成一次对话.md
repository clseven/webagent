# 创建会话并完成一次对话

## 1. 创建会话

1. 前端调用 `POST /api/sessions`，可选传 `appId`。
2. `ConversationServiceImpl.createSession` 创建 `ConversationSessionEntity`，生成 UUID 作为 sessionId，记录 userId、标题（默认"新对话"）、appId 和 enabledSkillIds。
3. `CompletableFuture.runAsync` 异步创建沙箱，不阻塞前端响应。
4. 如果数据库已有沙箱记录（`hasSandbox`），跳过创建。
5. 前端把新会话设为当前会话，加载历史（首次为空）。

## 2. 发送消息

### 2.1 请求阶段

1. Chat 页面调用 `api.createChatStream`，访问 `GET /api/sessions/{id}/chat/stream`。
2. `AgentController` 设置请求级搜索/规划开关（`UserContext`）。
3. `AgentServiceImpl.chatStream` 校验会话归属（`validateSessionOwnership`）。

### 2.2 上下文准备

`AgentTurnContextService.prepare` 的完整步骤：

1. 加载最近 20 条历史消息。
2. 调用 `judgeIntent` 分类本轮意图（SOCIAL/TASK/AMBIGUOUS），产出 [[#TurnPolicy]]。
3. 判断是否跳过规划（用户开关 + 轻量路由）。
4. 确保沙箱就绪（首次访问时同步创建）。
5. 记录是否首轮。
6. 用户消息入库。
7. 提取文件上下文（消息含"上传的文件"时拼出文件清单）。
8. 根据 policy 决定是否注入技能提示、工作区目录记忆、知识库增强。
9. 加载 Agent 应用，构建知识库描述。
10. 根据 policy 构建工具上下文（SOCIAL 用 empty）。
11. 汇总 systemPrompt，拼接顺序：文件上下文 → 知识库增强 → 工作区记忆 → 技能提示。
12. 返回 [[#AgentTurnContext]]。

### 2.3 规划阶段

1. `AgentPlannerService.plan` 检查 policy.shouldPlan()，SOCIAL 跳过返回 null。
2. 构造 `PlanAgent`，调用一次 LLM 产出结构化任务模型。
3. 规划输出有结构化校验（四个必须段落 + 协议标记拦截），两次失败走 fallback。
4. 发送 `plan` SSE 事件给前端。
5. 检查 `sink.isCancelled()`，用户中断时发送 `interrupted` 事件。

### 2.4 执行阶段

1. `ReactAgentFactory.createForStream` 创建执行器，SOCIAL 创建无工具执行器。
2. `ReactAgentHookService.configureForStream` 注册 Hook（logHook → fileStateCheck → viewImageHook → largeOutputHook → finalTodoGuard → 并发开关 → 子代理注入）。
3. `ReactAgent.runStream` 执行 ReAct 循环，发送 thinking、tool_call、observation、answer 事件。
4. 工具调用优先原生 tool_calls，回退 ReAct 文本解析。
5. PostToolUseHook 注入额外消息（如图片观察）。
6. StopHook 检查 TodoState 闭环。

### 2.5 后置处理

1. `done` 事件携带 tokenUsage，记录 executor token。
2. `AgentToolContextService.clearRuntimeState` 清理工具运行态（在 `doOnComplete` 和 `doOnError`）。
3. 首轮异步生成标题（`scheduleGeneratedTitle`）。
4. 前端把流式状态转成可恢复的历史消息。

## 3. 中断和异常

- 用户停止时，流式路径取消请求，保存 partial thinking/answer。
- `ReactAgent` 取消当前会话的后台任务。
- 异常发送 error SSE 事件，清理工具运行态。
- 流式结束、异常和取消路径都调用 `clearRuntimeState`。
- 外层 try-catch 兜底：contextRef 还没设置就异常时跳过清理。

## 4. 前端恢复

历史消息带有 plan、thinking、toolResult 等事件列表（`ChatMessageEntity.events`）。前端重新进入会话时可恢复工具行动流和产物卡片，而不是只显示最终文本。`AgentEventMapper.fromPlanAndSteps` 把规划文本和执行步骤映射为事件列表。

## 5. 同步路径差异

同步 `chat` 和流式 `chatStream` 共享上下文准备和执行器创建逻辑，区别：

- 同步从 `AgentResponse.getTotalUsage()` 一次记录 token，流式从 `done` 事件解析。
- 同步用 `conversationService.addAssistantMessage` 保存，流式由执行器内部保存。
- 同步在 `finally` 清理上下文，流式在 `doOnComplete`/`doOnError` 清理。
- 同步无中断检测，流式有 `sink.isCancelled()`。

## 6. 术语速查

### TurnPolicy

单轮对话策略开关，SOCIAL 对应 LITE（全 false），TASK/AMBIGUOUS 对应 FULL（全 true）。控制本轮是否注入规划、工具、技能、工作区、知识库和 StopHook。

### AgentTurnContext

单轮对话上下文聚合对象，把历史、用户消息、策略开关、用户 ID、应用、系统提示、工具上下文统一传给规划和执行阶段。
