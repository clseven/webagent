# 创建会话并完成一次对话

## 创建会话

1. 前端调用 `POST /api/sessions`。
2. 后端创建 `ConversationSessionEntity`，记录 userId、标题、应用 ID 和启用 Skill 集合。
3. 会话创建后可异步确保用户级 Sandbox 存在。
4. 前端把新会话设为当前会话，并加载历史。

## 发送消息

1. Chat 页面调用 `api.createChatStream`，访问 `GET /api/sessions/{id}/chat/stream`。
2. `AgentController` 设置请求级搜索/规划开关。
3. `AgentServiceImpl.chatStream` 校验会话归属。
4. `AgentTurnContextService.prepare` 准备历史、上传文件说明、应用配置、知识库上下文、Skill prompt 和工具上下文。
5. `AgentPlannerService` 生成计划，后端发送 `plan` SSE 事件。
6. `ReactAgentFactory` 创建执行器，`ReactAgentHookService` 注册 Hook。
7. `ReactAgent.runStream` 执行 ReAct 循环并发送 thinking、tool_call、observation、answer 等事件。
8. 完成后保存 assistant 消息、事件列表和 token 用量。
9. 前端把流式状态转成可恢复的历史消息。

## 中断和异常

- 用户停止时，流式路径会取消请求并保存 partial thinking/answer。
- `ReactAgent` 会取消当前会话的后台任务。
- 异常会发送 error SSE 事件，并清理工具运行态。
- 流式结束、异常和取消路径都应调用 `AgentToolContextService.clearRuntimeState`。

## 前端恢复

历史消息带有 plan、thinking、toolResult 等事件列表。前端重新进入会话时可恢复工具行动流和产物卡片，而不是只显示最终文本。
