# 知识库文档上传与检索

## 上传

1. 用户在 Knowledge 页面选择知识库并上传文档。
2. `RagController` 校验知识库归属。
3. `KnowledgeServiceImpl` 保存原始文件到用户级知识库目录。
4. 文档解析服务提取文本。
5. `TextSplitterService` 切片。
6. 切片元数据保存到 MySQL。
7. Embedding 服务批量生成向量。
8. Milvus 启用时写入向量；关闭时 Noop 服务跳过。
9. 文件同步到用户 Sandbox 的 `/home/gem/knowledge/{kbId}`。

## 检索

1. 用户或 Agent 发起查询。
2. Query Rewrite 生成多个检索 query。
3. 每个 query 做 Embedding。
4. 对目标知识库执行向量召回。
5. 合并并按 docId/chunkIndex 去重。
6. BGE Rerank 按原始问题排序。
7. 返回 topK 切片给前端，或格式化为 Agent 上下文。

## Agent 使用

应用关联知识库后，`knowledge_search` 工具会带有默认知识库范围。Agent 不应凭空回答知识库问题，而应先检索再回答。知识增强失败时主对话降级为空上下文，不中断。

## 预览

知识库文档可通过 `GET /api/rag/document/{docId}/file` 读取原文，通过 `GET /api/rag/document/{docId}/chunks` 读取切片。`FilePreviewer` 会把原文和切片并排展示。
