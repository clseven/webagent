# 知识库文档上传与检索

## 1. 上传

### 1.1 同步阶段

入口方法：`RagController.upload` → `KnowledgeServiceImpl.upload`

1. 校验知识库归属（`requireOwnedKnowledgeBase`）。
2. 清洗文件名（`sanitizeFileName`）。
3. 同名检查（忽略大小写），命中抛 `DuplicateFileException`，不进入异步处理。
4. 创建 `KnowledgeDocumentEntity`，状态 `PENDING`。
5. 保存原始文件到用户级知识库目录。
6. 触发异步处理，立即返回 `PENDING`。

### 1.2 异步阶段

入口方法：`KnowledgeDocumentProcessor.processDocumentAsync`（`@Async("knowledgeTaskExecutor")`）

1. 状态改为 `PROCESSING`。
2. `DocumentParserServiceImpl`（基于 Apache Tika）解析文档提取文本。
3. `TextSplitterService` 按 `smart/custom` 切片并回填原文偏移。
4. 如果没有生成任何切片，状态改 `FAILED`，错误信息"文档内容为空或无法划分"。
5. `EmbeddingService.embedBatch` 批量生成向量，获取真实 `promptTokens`。
6. 保存切片到 MySQL（每片含正文、序号、tokenCount、偏移）。
7. `VectorStoreService.insertBatch` 写入 Milvus；关闭时 Noop 跳过。
8. `syncToSandbox` 同步文件到用户 Sandbox 的 `/home/gem/knowledge/{kbId}`。
9. 沙箱同步成功时异步触发 Office 预转换。
10. 更新 `chunkCount`、`totalTokens`、`sandboxSynced` 和状态 `READY`。
11. 任意异常状态改 `FAILED`，异常消息写入 `errorMsg`。

详细规则见 [[文档摄取与切片]]。

### 1.3 前端轮询

前端 `Knowledge.js` 在上传完成后启动轮询（每 3s 调 `loadDocuments`），直到所有文档稳定为 `READY`/`FAILED` 才停止。组件卸载时清理定时器。

## 2. 检索

入口方法：`KnowledgeEnhancerImpl.enhance` 或 `KnowledgeServiceImpl.search`

1. Query Rewrite 生成多个检索 query（默认 DeepSeek，JSON mode）。
2. 每个 query 做 Embedding。
3. 对目标知识库执行并行向量召回（`topN=20`）。
4. 合并并按 `docId + chunkIndex` 去重，保留最高分。
5. 过滤低于 `minScore=0.5` 的候选。
6. BGE Rerank 按原始问题排序，截取 `topK=8`。
7. 补充文档名和切片正文。
8. 返回 topK 切片给前端，或格式化为 Agent 上下文。

详细规则见 [[查询改写与重排]]。

## 3. Agent 使用

应用关联知识库后：

1. `AgentKnowledgeContextService.buildKnowledgeDescription` 把知识库描述注入 `knowledge_search` 工具描述。
2. `AgentKnowledgeContextService.enhance` 调用 `KnowledgeEnhancer.enhance` 生成增强上下文。
3. `KnowledgeSearchTool` 的默认 kbId 取第一个关联知识库。
4. Agent 不应凭空回答知识库问题，而应先检索再回答。
5. 知识增强失败时主对话降级为空上下文，不中断。
6. SOCIAL 轮次跳过知识库增强（`policy.shouldInjectKB()` 为 false）。

## 4. 预览

| 接口 | 路径 | 说明 |
| --- | --- | --- |
| 获取切片 | `GET /api/rag/document/{docId}/chunks` | 返回全部切片（含序号、正文、tokenCount、偏移） |
| 获取原文 | `GET /api/rag/document/{docId}/file` | 从沙箱读取原始文件返回预览 |

`getPreviewContent` 在读沙箱前会调 `ensureSandboxFile` 按需恢复：沙箱缺文件时从本地补传。`getFileContent` 不做按需恢复，沙箱文件缺失即报错。

`FilePreviewer` 会把原文和切片并排展示。Office 文件走预转换链路，见 [[Office 文件预览]]。

## 5. 风险与排查

| 现象 | 优先检查 | 说明 |
| --- | --- | --- |
| 上传后一直处理中 | `status`、异步线程池、Embedding API | 前端每 3s 轮询 |
| 文档 FAILED | `errorMsg`、Tika 解析、Embedding Key | 失败前可能已写入切片和向量 |
| 检索为空 | `rag.milvus.enabled` | Milvus 默认关闭 |
| 预览不可用 | 本地 `storagePath`、`sandboxSynced` | `/file` 接口会按需从本地补传 |
| READY 但预览不可用 | `sandboxSynced` 是否 false | READY 不等于沙箱已同步 |
| 同名上传报错 | 同一知识库下同名文件 | 抛 DuplicateFileException，走替换接口 |
