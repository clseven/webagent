# 变更 2026-07-14 子代理知识库补充

## 背景

子代理（run_subagent）和后台任务系统已在代码中实现（`RunSubagentTool`、`BackgroundTaskManager`、`SubAgentConfigProperties`），但知识库中只有零散提及（[[计划与 ReAct 执行]] 第 8 节、[[工具目录]] 工具列表、[[工具系统模块]] 工具来源），缺少独立的模块页面来完整描述架构、上下文隔离、执行模式和后台任务管理。本次补充新建了 [[子代理与后台任务]] 模块页。

## 本次覆盖的代码事实

### 子代理

- `RunSubagentTool` 实现 `Tool` 接口，工具名 `run_subagent`，是标准工具，ReactAgent 循环不需任何修改。
- 子代理从父 ReactAgent fork，拥有独立 `messages[]`，完成后只回传结构化摘要，不污染主会话上下文。
- 子代理禁止递归创建子代理（`run_subagent` 工具自身被排除）。
- 子代理继承父 Agent 的 PreToolUse/PostToolUse Hook 和本轮 AgentTimeContext 时间快照。
- 子代理类型分 `analyzer`（10 轮）、`searcher`（5 轮）、`browser`（10 轮）、`general`（15 轮）。
- 类型只影响系统提示词，不再维护独立工具白名单。
- `agent.sub-agent.enabled` 默认 true，`agent.sub-agent.timeout-seconds` 默认 120 秒。

### 后台任务

- `BackgroundTaskManager` 使用固定 5 线程池 + daemon + Semaphore(5) 限流。
- 后台任务通过 `Supplier<String>` 闭包传递，超时和异常在调用方包好。
- 完成后结果摘要（截断 300 字）包装为 `<task_notification>` 投入无锁通知队列。
- 主循环 `collect()` 拉取通知注入为 user 消息，`awaitPending()` 阻塞等待遗留任务。
- 并发超限静默回退同步执行，`cancelAll()` 通过 `Future.cancel(true)` 真取消。
- 当前只在同步路径 `chat()` → `run()` 支持后台，流式路径不支持（三期）。

### 父 Agent 注入

- `ReactAgentHookService.wireSubAgentParent` 在 Hook 装配阶段把当前 ReactAgent 注入 RunSubagentTool。
- 同步和流式路径各调用一次。
- 后台模式下闭包捕获具体 child 实例，不存在 singleton parentAgent 风险。

## 已知问题

1. `SubAgentConfigProperties.maxIterations` 目前未生效，子代理仍走 `ReactAgent.MAX_ITERATIONS = 25`。
2. LLM 启动后台子代理后偶尔会自己重复做同样的搜索，提示词需优化。
3. 多 tool_call 时第二个被丢弃（单 tool_call per turn 限制）。
4. `runStream()` 未支持后台任务（三期）。
