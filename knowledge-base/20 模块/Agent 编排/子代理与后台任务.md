# 子代理与后台任务

## 1. 模块概览

本文说明子代理（SubAgent）和后台任务系统的设计原理、触发机制、上下文隔离策略和配置方式。子代理让主 Agent 能将复杂子任务委托给独立的子 ReactAgent 执行，完成后只回传结构化摘要；后台任务系统让慢操作不阻塞主循环。

当前实现里需要特别注意七点：

- 子代理 = 标准工具。`run_subagent` 遵循和 `read_file`、`execute_command` 完全一致的调用路径，ReactAgent 循环不需任何修改。
- 子代理拥有独立 `messages[]`，其内部工具调用的原始输出不进入主会话上下文，完成后只返回摘要。
- 子代理从父 ReactAgent fork，继承 PreToolUse/PostToolUse Hook，安全策略不跳过。
- 子代理禁止递归创建子代理（`run_subagent` 工具自身被排除）。
- 子代理可同步或后台执行；后台执行时由 `BackgroundTaskManager` 管理，主循环后续轮次收到完成通知。
- 后台任务通过 `Supplier<String>` 闭包传递，超时和异常已在调用方包好，消除 RunSubagentTool singleton parentAgent 风险。
- 后台任务并发上限由 `agent.background.max-concurrent` 控制，超限静默回退同步执行。

## 3. 架构概览

```text
主 Agent（ReactAgent）
  │
  ├─ LLM 输出 tool_call: run_subagent
  │
  ├─ RunSubagentTool.execute()
  │    ├─ 检查类型配置（agent.sub-agent.types.*）
  │    ├─ 获取受限工具列表（排除 run_subagent 自身）
  │    ├─ 获取子代理系统提示词（按类型）
  │    ├─ parentAgent.fork(restrictedTools, prompt) → 子 ReactAgent
  │    │
  │    ├─ run_in_background=false → 同步执行（带超时）
  │    │    └─ child.run(sessionId, task, List.of())
  │    │         └─ 独立 ReAct 循环 → 返回 AgentResponse
  │    │
  │    └─ run_in_background=true → 后台执行
  │         └─ backgroundTaskManager.start(sessionId, 闭包)
  │              └─ 占位 tool_result → 主循环继续
  │              └─ 子代理在后台线程跑完 → 通知入队
  │
  ├─ 后续轮次：collect() → 有通知 → 注入 messages
  │
  └─ 最终回答前：awaitPending() → 阻塞等待遗留任务
```

## 4. 子代理类型

| 类型 | 职责 | max-iterations | 系统提示词特点 |
| --- | --- | --- | --- |
| `analyzer` | 数据分析（读文件 + 分析） | 10 | 要求输出分析摘要（数据概况、关键发现、问题列表、建议），不超过 500 字 |
| `searcher` | 知识库检索 | 5 | 要求输出检索摘要（查询分析、相关内容、综合回答），标注来源 |
| `browser` | 浏览器操作 | 10 | 要求输出操作摘要（执行操作、关键结果），截图后必须调 view_image 再分析 |
| `general` | 通用任务 | 15 | 要求输出执行摘要（完成操作、关键结果、产出文件、后续建议） |

每种类型可单独开关和设最大迭代数，由 `agent.sub-agent.types.*` 控制。类型只影响子代理提示词，不再维护独立工具白名单。

## 5. 上下文隔离

子代理启动时是一张白纸，不继承主会话的：

- 历史对话
- 工具调用记录
- 中间状态

子代理的上下文组成：

| 上下文组成部分 | 来源 |
| --- | --- |
| 独立系统提示词 | 子代理类型专属 |
| 任务指令 | 主 Agent 传递，描述具体任务 |
| 工具列表 | 从父 Agent 工具集过滤（排除 run_subagent） |
| 执行记录 | 子代理自己调用工具产生，不返回给主 Agent |
| 中间输出 | 不返回给主 Agent |

关键约束：

- 子代理继承父 Agent 的本轮 [[AgentTimeContext]] 时间快照，不重新取时钟。
- 子代理的 `sessionId` 从父 Agent 传入，但 `conversationService` 传 null，不写主会话历史。
- 子代理完成时只返回 `AgentResponse.getFinalAnswer()` 作为摘要。

## 6. 工具限制策略

子代理的工具列表来自父 Agent 当前可用工具集，过滤规则：

1. 遍历父 Agent 的全部工具。
2. 永远排除 `run_subagent` 自身（禁止递归创建子代理）。
3. 其余工具全部保留。

不再按类型维护独立白名单，防止新增工具遗漏。子代理类型只影响系统提示词，不影响工具集。如果后续需要细粒度权限控制，应回到按类型过滤的方式。

## 7. 父 Agent 注入

入口方法：`ReactAgentHookService.wireSubAgentParent`

在 Hook 装配阶段（`configureForChat` 和 `configureForStream`），遍历工具列表找到 `RunSubagentTool`，把当前 `ReactAgent` 作为父 Agent 注入。子代理通过父 Agent 引用 fork 出受限工具集。

关键约束：

- 同步和流式路径各自装配一次，漏掉一条路径会导致子代理工具不可用。
- `RunSubagentTool` 是 Spring 单例，但 `parentAgent` 字段是运行时注入的实例引用，每轮对话设置一次。
- 后台执行模式下，闭包捕获的是具体的 `child` 实例，不存在 singleton parentAgent 被其他会话覆盖的风险。

## 8. 执行模式

### 8.1 同步执行

子代理在主 Agent 的工具调用中同步执行，主 Agent 等待子代理完成后继续。

- 通过 `CompletableFuture.supplyAsync` 在异步线程执行 `child.run()`。
- 使用 `.orTimeout(timeoutSeconds, TimeUnit.SECONDS)` 设置超时，默认 120 秒。
- 超时返回中文超时提示，不抛异常。
- 执行失败返回错误消息，不抛异常。
- 完成后返回 `AgentResponse.getFinalAnswer()` 作为工具结果。

### 8.2 后台执行

当 LLM 在 `run_subagent` 参数中设置 `run_in_background=true` 时，子代理在后台执行，主 Agent 不等待。

- `BackgroundTaskManager.start()` 提交闭包到线程池，返回 `bg_id`。
- 工具立即返回占位结果 `[后台子代理 bg_xxx 已启动]`，主循环继续。
- 子代理在后台线程跑完后，结果摘要（截断 300 字）包装为 `<task_notification>` 投入通知队列。
- 主循环每轮迭代开始时调用 `collect()` 拉取通知，注入为 user 消息。
- 最终回答前调用 `awaitPending()` 阻塞等待所有遗留任务（最多 6 轮消化循环）。

### 8.3 何时使用子代理

| 场景 | 是否需要子代理 | 原因 |
| --- | --- | --- |
| 简单单工具调用 | 否 | 直接执行即可 |
| 顺序依赖明确的任务 | 否 | 主 Agent 直接执行 |
| 任务可并行 | 是 | 多个子代理同时执行 |
| 任务复杂、工具调用多 | 是 | 子代理独立上下文，不污染主会话 |
| 执行复杂 Skill | 是 | Skill 内部逻辑封装在子代理里 |

## 9. 后台任务管理器

`BackgroundTaskManager` 是统一的后台任务调度器，管理慢操作的后台执行。

### 9.1 线程模型

- `newFixedThreadPool(5)` + daemon 线程，复用不销毁。
- `Semaphore(5)` 做准入限流，超限静默回退同步执行（`start()` 返回 null）。
- 并发上限由 `agent.background.max-concurrent`（默认 5）控制。

### 9.2 通知机制

- `ConcurrentLinkedQueue<String>` 无锁通知投递，按 sessionId 隔离。
- 后台任务完成（或失败）后，结果摘要包装为 `<task_notification>` 格式投入队列。
- 通知格式：`<task_notification><task_id>bg_xxxx</task_id><status>completed/failed</status><summary>...</summary></task_notification>`。
- 主循环主动 `poll`，不依赖回调。
- `collect()` 拼接所有待处理通知为一条消息。

### 9.3 取消策略

- `Future.cancel(true)` + interrupt 实现真取消，不只删引用。
- `cancelAll(sessionId)` 取消该会话所有后台任务，清理通知队列和计数。
- SSE 中断时由 `doOnError`/`doOnCancel` 触发 `cancelAll`。

### 9.4 闭环机制

- `awaitPending(sessionId, timeoutMs)` 阻塞等待所有未完成任务，返回期间收集的通知。
- 主 Agent 出循环后调用 `awaitPending`，有遗留通知则注入并进入消化循环（最多 6 轮）。
- 消化循环中 LLM 看到通知后可能继续调用工具，需要多轮消化。
- `compressIfNeeded` 跳过含 `<task_notification>` 的消息，通知才几百字不需要压缩。

### 9.5 流式路径限制

当前只在同步路径 `chat()` → `run()` 实现了后台任务。流式路径 `chatStream()` → `runStream()` 传给 `BackgroundTaskManager` 的引用为 null，后台功能不生效。三期再做。

## 12. 风险与排查

| 现象 | 优先检查 | 说明 |
| --- | --- | --- |
| `run_subagent` 工具不出现 | `agent.sub-agent.enabled`、`AgentToolContextService.filterSubAgentTool` | 开关关闭时工具被移除 |
| 子代理超时 | `agent.sub-agent.timeout-seconds`、任务复杂度 | 默认 120 秒，超时返回提示不阻断主流程 |
| 子代理未返回有效结果 | `AgentResponse.getFinalAnswer()` 是否为空 | 子代理可能未按提示词格式输出摘要 |
| 后台子代理通知未注入 | `BackgroundTaskManager.collect()`、`awaitPending()` | 检查通知队列是否被正确 poll |
| 后台并发已满 | `agent.background.max-concurrent`、Semaphore | 超限回退同步，日志会有"并发已满" |
| 后台任务残留 | `cancelAll()` 是否在 `doOnError`/`doOnCancel` 调用 | SSE 中断后未清理 |
| 流式路径后台不生效 | `chatStream()` 是否传 `BackgroundTaskManager` | 当前流式路径不支持后台，三期 |
| LLM 重复执行子代理任务 | 提示词未明确"启动后台后不要管" | 已知问题，提示词需优化 |
| 多 tool_call 第二个被丢弃 | ReactAgent 单 tool_call per turn 限制 | 已知限制，非子代理独有 |
| 子代理 max-iterations 未生效 | `SubAgentConfigProperties.maxIterations` | 已知问题，子代理仍走 `ReactAgent.MAX_ITERATIONS = 25` |

## 13. 术语速查

本节用于 Obsidian 悬浮预览。阅读正文时，把鼠标停在带链接的术语上，可以快速看到这里的释义。

### RunSubagentTool

`RunSubagentTool` 是子代理工具，工具名 `run_subagent`。通过 `wireSubAgentParent` 把父 `ReactAgent` 注入，让子代理能 fork 出受限工具集执行子任务。可同步或后台运行，受 `agent.sub-agent.enabled` 控制。禁止递归创建子代理。

### SubAgentConfigProperties

`SubAgentConfigProperties` 是子代理配置属性，控制 `run_subagent` 工具中各子代理类型的行为：总开关、超时和各类型（analyzer/searcher/browser/general）的 enabled 和 max-iterations。

### BackgroundTaskManager

`BackgroundTaskManager` 管理后台子代理任务和慢操作，最大并发由 `agent.background.max-concurrent` 控制。主循环后续轮次从它收集完成结果并注入对话。使用固定线程池 + Semaphore 限流 + Future 真取消 + 无锁通知队列。

### fork

`fork` 是 `ReactAgent` 的方法，从父 Agent 创建子 ReactAgent 实例。子代理继承父 Agent 的 Hook 和时间快照，但拥有独立的 `messages[]`，不继承主会话历史。工具列表按参数过滤。

### wireSubAgentParent

`wireSubAgentParent` 是 `ReactAgentHookService` 的方法，在 Hook 装配阶段把当前 `ReactAgent` 作为父 Agent 注入到 `RunSubagentTool`。同步和流式路径各调用一次。

### task_notification

`task_notification` 是后台任务完成后的通知 XML 格式，包含 task_id、status 和 summary。主循环通过 `collect()` 拉取并注入为 user 消息，让 LLM 知道后台任务的结果。`compressIfNeeded` 跳过含此标记的消息。
