# 认证与用户模块

## 职责

认证与用户模块负责注册、登录、当前用户识别和请求级用户上下文。系统的大多数业务数据都按 userId 隔离：会话、应用、知识库、上传目录、用户 Sandbox、用户动态 MCP 配置等。

## 接口

- `POST /api/auth/register`。
- `POST /api/auth/login`。
- `GET /api/auth/me`。

前端把 token、用户名和当前会话 ID 存在 `localStorage`。路由守卫通过 `/api/auth/me` 校验登录状态。

## 用户上下文

`UserContext` 保存当前请求用户 ID，并带有 WebSearch、Planning 等请求级开关。Controller 和 Service 在处理会话、知识库、文件、Sandbox 时都应先确认数据属于当前用户。

## 与 Sandbox 的关系

当前 Sandbox 资源按用户而不是会话持有。`SandboxServiceImpl` 通过会话解析 userId，再查找或创建用户级 Sandbox。沙箱视图 token 也只绑定 userId，并在代理请求时实时查当前 endpoint。

## 风险提醒

多租户边界是面试和代码审查重点。新增接口时要明确：

- 是否从当前用户上下文取 userId。
- 是否校验目标资源属于该用户。
- 是否可能通过文件路径、docId、sessionId、appId 越权。
