# 文档摄取与切片

## 上传流程

知识库文档上传由 `RagController` 和 `KnowledgeServiceImpl` 处理：

1. 校验知识库属于当前用户。
2. 保存原始文件到用户级知识库目录。
3. 使用文档解析能力抽取文本。
4. 按配置进行切片。
5. 保存文档和切片元数据到 MySQL。
6. 调用 Embedding 服务生成向量。
7. 如果 Milvus 启用，将向量写入集合；否则由 Noop 向量服务跳过。
8. 同步知识库文件到用户 Sandbox 的 `/home/gem/knowledge/{kbId}`。

## 切片规则

默认 chunk size 是 500，overlap 是 50。切片需要保留足够上下文，同时不能过长影响检索和 rerank 成本。

## 文件迁移

`KnowledgeFileMigrationService` 负责把旧路径中的知识库文件迁移到用户级规范目录，并在校验大小后更新数据库。这个迁移会在 Sandbox 创建和工作区恢复前运行，避免旧文件路径造成预览或同步失败。

## 替换和删除

- 替换文档会重新解析、切片、Embedding 和向量写入。
- 删除文档会删除 MySQL 元数据，并调用向量服务删除对应 docId。
- 删除知识库会删除该知识库下文档和向量。
- Milvus 关闭时，向量删除由 Noop 实现记录跳过。
