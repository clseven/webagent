# 查询改写与重排

## Agent 检索增强

`KnowledgeEnhancerImpl` 面向 Agent 对话增强。流程是：

1. Query Rewrite 根据用户问题和历史生成多个检索查询。
2. 对每个查询生成 Embedding。
3. 对应用关联的知识库执行向量检索。
4. 合并候选切片并去重。
5. 调用 Rerank 服务按原始用户问题排序。
6. 取 topK 切片格式化为上下文，注入 Agent。

## Knowledge 页面搜索

`KnowledgeServiceImpl.search` 也使用类似流程：query rewrite、多查询向量召回、去重、rerank、返回切片结果。前端 Knowledge 页面可用于直接验证某个知识库的检索效果。

## Query Rewrite

默认实现调用 Ollama chat API：

- URL：`RAG_REWRITE_URL`，默认 `http://localhost:11434/api/chat`。
- 模型：`qwen3:1.7b`。
- maxQueries：3。
- 低温度、较短输出，目标是生成检索语句，不是回答用户。

关闭 rewrite 时，应退化为原始 query。

## Rerank

默认实现调用 BGE rerank API：

- URL：`RAG_RERANK_URL`，默认 `http://127.0.0.1:8002/rerank`。
- topK：8。
- 服务失败时退化为按原始分数排序。

## 当前风险

如果 Milvus 默认关闭，检索阶段会返回空候选；这种配置适合开发启动，但不适合演示 RAG 效果。面试或演示前需要确认 `RAG_MILVUS_ENABLED=true` 且集合已有向量数据。
