# 浏览器工具

## 1. 工具组合

浏览器工具围绕 AIO Sandbox 中的 Browser Agent 运行时（`/home/gem/.runtime/browser-agent`）：

| 工具名 | 功能 | sideEffect |
| --- | --- | --- |
| `browser_info` | 确认浏览器代理状态 | READ |
| `browser_inspect` | 读取页面文本、DOM 和结构化信息 | READ |
| `browser_execute` | 执行复杂浏览器脚本、导航和等待 | EXCLUSIVE |
| `browser_action` | 坐标点击、滚动、快捷键等视觉兜底动作 | EXCLUSIVE |
| `browser_screenshot` | 保存截图并返回下载链接 | READ |
| `view_image` | 读取图片文件，由视觉 Hook 调用 Agnes 观察图片内容 | READ |

Browser Agent 运行时在沙箱创建时由 `installBrowserAgentRuntime` 安装，包含 `browser-agent.mjs`、`package.json` 和 `npm install playwright-core`。安装失败只禁用浏览器能力，不阻断其他能力。

## 2. 截图交付规则

`browser_screenshot` 的 `deliver_to_user` 参数：

| 值 | 使用场景 |
| --- | --- |
| `false`（默认） | 过程截图、验证码、拦截页、加载失败页、仅用于判断的截图 |
| `true` | 用户最终要看的截图 |

前端会过滤 `browser_screenshot` 产物，只有 `deliverToUser === true` 的截图进入图片卡片组。这避免模型在浏览过程中把大量临时截图误交付给用户。

## 3. 图片观察规则

截图或生成图片后，如果 Agent 需要理解图片内容，应继续调用 `view_image`：

1. `view_image` 工具读取图片文件，把字节和元数据放入 `ImageBuffer`。
2. `ReactAgentHookService.viewImageHook`（PostToolUseHook）从 `ImageBuffer.take(sessionId)` 取出图片。
3. 构造 vision 消息，调用 `visionLlm`（Agnes `agnes-2.0-flash`）生成客观观察文本。
4. 观察文本作为 user 消息注入主执行模型。
5. 视觉模型失败时降级为"图片已加载但无法分析"提示，不阻断主流程。

## 4. 前端展示

Chat 页会把正式图片产物聚合成图库：

- 最多先展示 4 张缩略图。
- 多余图片显示隐藏数量。
- 点击后进入 `FilePreviewer` 图片 lightbox，可在同一组图片中前后切换和缩放。

## 5. 并发调度

- `browser_info`、`browser_inspect`、`browser_screenshot`、`view_image` 为 READ，可并发。
- `browser_execute`、`browser_action` 为 EXCLUSIVE，完全串行（浏览器操作有全局状态）。

## 6. 风险与排查

| 现象 | 优先检查 | 说明 |
| --- | --- | --- |
| 浏览器工具不可用 | `/home/gem/.runtime/browser-agent` 是否安装 | 安装失败只禁用浏览器能力 |
| browser_info 无响应 | Browser Agent 运行时、playwright-core 依赖 | 检查 npm install 日志 |
| view_image 无观察结果 | `ImageBuffer`、`visionLlm` 配置、`AGNES_API_KEY` | 视觉模型失败降级为提示 |
| 截图不展示 | `deliver_to_user` 是否为 true | 只有 true 的截图进入卡片组 |
| 浏览器操作并发冲突 | sideEffect 标注 | execute 和 action 为 EXCLUSIVE，不会并发 |
