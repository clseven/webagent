# 文件与文档工具

## 1. 文件操作工具

文件工具面向 AIO Sandbox 工作区，通过 `AioClient.files()` 操作 `/home/gem` 下的文件。

| 工具名 | 功能 | sideEffect | sandboxType |
| --- | --- | --- | --- |
| `list_files` | 列目录 | READ | AIO |
| `read_file` | 读文件内容（按行窗口分页） | READ | AIO |
| `write_file` | 写文件（创建或覆盖） | WRITE | AIO |
| `file_search` | 搜索文件（grep/find） | READ | AIO |
| `file_replace` | 简单文本替换 | WRITE | AIO |
| `str_replace_editor` | 结构化文本编辑（支持 view/create/str_replace/insert） | WRITE | AIO |
| `download_file` | 生成可交付的文件下载链接 | READ | AIO |

这些工具通常操作 `/home/gem/workspace`、`/home/gem/uploads` 或 `/home/gem/output` 目录。

## 2. 文档处理工具

| 工具名 | 功能 | sideEffect | sandboxType |
| --- | --- | --- | --- |
| `parse_document` | 调用沙箱内 `file_parser.py` 从 PDF/Office 抽取文本 | READ | AIO |
| `convert_to_markdown` | 把文档转换为 Markdown | READ | AIO |

`parse_document` 调用沙箱内 `/home/gem/tools/file_parser.py`，支持 PDF/DOCX/XLSX/PPTX。工具脚本在沙箱创建时由 `uploadToolScript` 上传。知识库上传链路仍由 [[RAG 知识库模块]] 管理，不应混同为普通文件工具。

## 3. 产物交付

`download_file` 表示"最终交付文件"。前端在同一条 assistant 消息中同时看到下载产物和截图产物时，会优先展示正式交付文件，避免重复展示临时截图或同一文件的旧式链接卡片。

## 4. 并发调度

文件工具按副作用类型调度：

- `list_files`、`read_file`、`file_search`、`parse_document`、`convert_to_markdown`、`download_file` 为 READ，可并发。
- `write_file`、`file_replace`、`str_replace_editor` 为 WRITE，串行且与 READ 互斥。

## 5. 与文件预览的关系

工具负责生成或读取文件；前端的统一预览由 [[Office 文件预览]] 和 [[文件与工作区模块]] 承担。Agent 如果要让用户看到文件，应使用 `download_file` 或产生可预览路径，而不是只在文本里描述路径。

## 6. 风险与排查

| 现象 | 优先检查 | 说明 |
| --- | --- | --- |
| 文件读写失败 | 沙箱是否可用、路径是否合法 | 工具返回错误消息字符串，不抛异常 |
| parse_document 失败 | `/home/gem/tools/file_parser.py` 是否存在 | 工具脚本在沙箱创建时上传，检查 `uploadToolScript` 日志 |
| 并发写冲突 | sideEffect 标注是否正确 | WRITE 类工具串行，不会与 READ 并发 |
| 下载链接失效 | token 是否过期、文件是否被删 | download_file 生成的链接有有效期 |

## 7. read_file 按行窗口分页

`read_file` 不再整本读入内存后截断，而是利用 AIO 文件接口原生的 `start_line`/`end_line` 做服务端分页，跟齐 Cline/Roo-Code 范式：

- 参数：`path`（必填）、`offset`（0 基，默认 0）、`limit`（默认 2000，上限 5000）。
- 调用 `client.files().readText(path, offset, offset + limit)`，AIO 只返回该窗口的行。
- 单行超过 `MAX_LINE_CHARS = 2000` 字符截断（防 minified/打包产物），单次总输出不超过 `MAX_READ_OUTPUT_CHARS = 48000`。
- observation 末尾带翻页提示：满窗口 → `[传 offset=Y 查看后续]`；到尾 → `[已显示到文件末尾]`；触总量上限 → `[传 offset=Y 查看后续]`。

这样大文件分页小读，每页远小于上下文阈值，不会单条撑爆上下文。其它一次性输出工具（shell/MCP/文档解析）分不了页，仍靠 `largeOutputHook`（见 [[Agent 工具调用]]），head+tail 截断是后续 TODO。
