# 工具系统模块

## 职责

工具系统把外部动作封装成 `Tool`：每个工具提供 `ToolDefinition` JSON Schema，执行时接收 `sessionId` 和参数 Map，并返回字符串 observation。ReactAgent 不关心工具内部如何访问沙箱、文件、MCP 或知识库，只把工具返回值写回模型消息。

## 工具来源

- 本地工具：`src/main/java/com/example/sandbox/web/service/tool` 下的 Java 实现。
- MCP 工具：`McpClientToolProvider` 从已连接 MCP Server 读取工具定义，再包装成 `RealMcpTool`。
- Skill 工具：`skill_list`、`skill_activate`、`skill_reference` 负责渐进式披露 Skill 内容。
- 运行时工具：`todo_write`、`run_subagent` 影响 Agent 执行闭环，但仍遵守 Tool 接口。

## 工具上下文

`AgentToolContextService` 在每轮对话前构建工具上下文：

- 根据当前用户、会话和应用准备可用工具。
- 结合 Sandbox 类型过滤工具。
- 将应用关联知识库写入 `knowledge_search` 默认上下文。
- 在 MCP 启用时把动态 MCP 工具合并到工具列表。
- 对话结束或异常时清理会话级运行状态。

## Hook 与 Tool 的边界

- Tool 本身只返回字符串，不直接追加 ChatMessage。
- 需要拦截工具调用时用 `PreToolUseHook`。
- 需要工具执行后注入额外模型可读消息时用 `PostToolUseHook`。
- 需要最终回答前强制检查时用 `StopHook`。

典型例子：`view_image` 工具只读取图片并放入 `ImageBuffer`；真正调用 Agnes 视觉模型的是 `ReactAgentHookService` 的 PostToolUseHook。

## 分类

- [[Shell 与生命周期工具]]：命令执行、等待、终止、沙箱申请。
- [[文件与文档工具]]：文件读写、替换、搜索、下载、文档解析和 Markdown 转换。
- [[浏览器工具]]：浏览器检查、执行、动作、截图和图片观察。
- [[Skill 与知识检索工具]]：Skill 披露、知识库检索、Web 搜索、MCP 管理。
- [[工具目录]]：面向阅读和排查的完整工具清单。
