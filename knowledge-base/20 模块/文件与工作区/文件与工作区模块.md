# 文件与工作区模块

## 用户级目录

本地根目录默认是 `./uploads`，当前规范结构为：

```text
uploads/users/{userId}/
├─ uploads/{fileName}
└─ knowledge/{kbId}/{fileName}
```

Sandbox 中对应：

```text
/home/gem/
├─ uploads/{fileName}
├─ workspace/
├─ output/
├─ skills/
├─ tools/
└─ knowledge/{kbId}/{fileName}
```

## 上传文件

普通会话文件由 `FileUploadController` 处理：

1. 校验会话归属并取得 userId。
2. 清理文件名中的路径片段。
3. 写入本地用户级 uploads 目录。
4. 上传到当前用户 AIO Sandbox 的 `/home/gem/uploads/{fileName}`。
5. 同名时返回 409，由前端选择替换或保留两份。
6. Office 文件触发异步预转换，便于后续预览。

## 下载、删除和路径查询

当前下载、删除和 `GET /api/files/path/{sessionId}` 都按会话所属 userId 访问用户级目录，不再依赖旧的会话目录存储。

- 下载会从 `uploads/users/{userId}/uploads/{safeFilename}` 读取。
- 删除会删除同一路径文件。
- 路径查询返回本地用户上传根目录和 Sandbox 挂载路径 `/home/gem/uploads`。

## 同步与恢复

- `FileSyncService` 把用户上传目录和知识库目录同步到新 Sandbox。
- `KnowledgeFileMigrationService` 迁移旧 storagePath 到用户级规范目录，并校验大小后更新数据库。
- `SandboxServiceImpl` 创建用户 Sandbox 后会先迁移旧知识文件，再恢复工作区，最后同步已启用 Skill。

## 前端工作区

`WorkspaceBrowser.js` 默认根目录是 `/home/gem`。它通过 `execute_command` 执行 `find`，构建树形结构；点击文件后调用 `FilePreviewer.preview({ source: 'workspace' })`。

当前 Chat 页面把工作区作为右侧工具 dock 的“文件”视图嵌入，不再只依赖独立浮动按钮。
