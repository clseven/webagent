# 系统边界

## 主要边界

| 边界 | 代表实现 | 负责 | 不负责 |
|---|---|---|---|
| Web/API 层 | `AgentController`、`SandboxController`、`FileUploadController`、`RagController` | 鉴权后的 HTTP/SSE 接入、参数接收、响应包装 | 直接执行 Agent 决策或复杂工具逻辑 |
| Agent 编排层 | `AgentServiceImpl`、`AgentTurnContextService`、`AgentPlannerService`、`ReactAgentFactory` | 会话校验、上下文准备、规划与执行串联、保存消息和 token | 直接实现工具、Skill、MCP 或 Hook 细节 |
| ReAct 执行层 | `ReactAgent` | 单轮工具选择、工具 observation 写回、Stop Hook、流式事件 | 业务持久化和外部契约细节 |
| Hook 层 | `ReactAgentHookService`、`FinalTodoGuardHook` | 拦截工具、注入额外消息、最终回答门禁（TodoState 前置硬信号 + 基于证据的模型自检） | 长业务流程编排 |
| Tool 层 | `Tool` 实现、`RealMcpTool` | 把确定动作封装成字符串 observation | 直接改写对话消息列表 |
| MCP 层 | `McpClientManager`、`McpConfigurationService`、`AioShellTransport` | 管理 MCP 连接、工具发现、调用、用户隔离 | 绕过工具系统直接暴露给 LLM |
| Sandbox 层 | `SandboxServiceImpl`、`AioClient`、`AioSandboxStore` | 用户级 AIO 生命周期、endpoint 恢复、文件/命令代理 | 前端直接访问内部 endpoint |
| 文件/RAG 层 | `UserWorkspaceStorageService`、`KnowledgeServiceImpl`、`VectorStoreService` | 用户文件、知识库原文、切片、向量检索 | Agent 规划和模型推理 |
| 前端层 | `Chat.js`、`FilePreviewer.js`、`WorkspaceBrowser.js` | 会话 UI、SSE 展示、工具 dock、文件预览 | 持久化业务规则 |

## 关键边界规则

- `AgentServiceImpl` 只做编排，不直接读写 Skill 文件、不直接管理 MCP 生命周期、不直接实现 Hook 逻辑。
- Tool 的 `execute()` 返回纯字符串，作为 observation 进入 ReAct 循环；需要注入图片观察、最终门禁等消息时用 Hook。
- `TodoState` 是执行中的看板和证据账本，不调度工具、不替代 PlanAgent。
- 沙箱资源按用户复用；`sessionId` 用于查用户和会话，不作为 AIO endpoint 的长期缓存 key。
- 浏览器沙箱视图只暴露同源 `/sandbox-view/{token}/...`，不把内部 `127.0.0.1:port` endpoint 直接交给浏览器。
- 用户动态 MCP 默认禁止 HTTP、私网和链路本地地址，除非配置显式放开。
- RAG 的 Milvus 是可选依赖；关闭时 `NoopVectorStoreServiceImpl` 接住向量写入和检索。
