# 系统运行方式

## 启动形态

项目是一个单体 Spring Boot 应用，默认端口 `8081`。前端静态文件、后端 API、SSE、沙箱视图代理都由同一个应用提供。

默认激活 profile：

```yaml
spring.profiles.active: deepseek,local
```

`application-deepseek.yml` 把 planner/executor 指向 DeepSeek；`application-agnes.yml` 可以把 planner/executor 切到 Agnes。无论 profile 怎么切，`visionLlm` 的默认配置仍来自 `application.yml` 的 Agnes 节点。

## 运行依赖

- MySQL：保存用户、会话、应用、知识库、文档、用户沙箱等元数据。
- AIO/OpenSandbox：提供用户级沙箱、shell、文件、浏览器、noVNC、code-server 等能力。
- DeepSeek：默认规划和执行模型。
- Agnes：默认视觉观察模型。
- 智谱 Embedding：默认知识库向量化模型。
- Milvus：默认关闭；启用后提供向量召回。
- Ollama：Query Rewrite 默认地址是 `http://localhost:11434/api/chat`。
- BGE Rerank：默认地址是 `http://127.0.0.1:8002/rerank`。

## 对话运行主线

1. 前端调用 `POST /api/sessions` 创建会话，并触发后端异步创建用户级 Sandbox。
2. 用户发送消息时，前端优先走 `GET /api/sessions/{id}/chat/stream` 的 SSE 流式接口。
3. `AgentTurnContextService` 准备历史、上传文件上下文、应用配置、Skill prompt、MCP 工具和普通工具。
4. `AgentPlannerService` 用执行模型生成计划。
5. `ReactAgentFactory` 创建执行器，`ReactAgentHookService` 注册 Hook 并给 `run_subagent` 注入父 Agent。
6. `ReactAgent` 执行工具循环，把 plan、thinking、tool_call、observation、answer 事件流式返回。
7. 完成后保存 assistant 消息、事件列表和 token 用量。

## 沙箱运行主线

- 应用启动时，`SandboxServiceImpl` 从 `user_sandbox` 恢复用户与 Sandbox 绑定，并通过 AIO 健康探针确认 endpoint。
- 新会话优先复用健康的用户 Sandbox，不健康则清理映射并重建。
- AIO endpoint 注册到用户级 key `__user_{userId}`，避免同一用户多个会话缓存出不同 endpoint。
- 浏览器视图通过 `GET /api/sessions/{id}/aio/view-url` 签发短期 token，再走 `/sandbox-view/{token}/...` 代理。

## 验证入口

- Java 单测：`mvn test`。
- 前端轻量 JS 测试：`node src/test/js/file-previewer-office.test.js`。
- 知识库校验：`powershell -ExecutionPolicy Bypass -File .agents/skills/maintain-project-knowledge/scripts/validate-knowledge-base.ps1`。
