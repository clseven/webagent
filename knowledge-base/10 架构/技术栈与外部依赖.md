# 技术栈与外部依赖

## 1. 后端技术栈

| 技术 | 版本 | 用途 |
| --- | --- | --- |
| Java | 17 | 运行时 |
| Spring Boot | 3.2.5 | 应用框架 |
| Spring Web | - | HTTP API 和静态资源 |
| Spring WebFlux | - | 模型、RAG、MCP 等 HTTP 客户端调用（WebClient） |
| Spring WebSocket | - | 沙箱视图同源代理转发 code-server、noVNC、Web Terminal 的 Upgrade 请求 |
| Spring Data JPA + MySQL | - | 业务元数据持久化 |
| Lombok | - | 简化实体和配置类 |
| Apache Tika | 2.9.1 | 知识库文档解析（PDF/DOCX/PPTX/XLS/HTML 等 700+ 格式） |
| Milvus Java SDK | 2.4.1 | 可选向量数据库 |
| MCP Java SDK | 2.0.0 | MCP Client、stdio/HTTP transport 和 JSON-RPC 类型 |
| Jsoup | - | WebSearchTool 解析搜索结果 |
| OpenSandbox SDK + AIO REST | - | 创建和操作沙箱 |

## 2. 前端技术栈

| 技术 | 用途 |
| --- | --- |
| 静态 Vue 3 + Vue Router | 前端框架，直接放在 `src/main/resources/static`，无独立构建步骤 |
| fetch + ReadableStream | 处理 SSE，需要携带 Authorization Header |
| Highlight.js CDN | 代码高亮 |
| FilePreviewer.js | 统一文件预览入口（图片、PDF、文本、Markdown、CSV、Office 转 PDF） |

## 3. 模型和智能体依赖

| 用途 | 模型 | 配置来源 | 说明 |
| --- | --- | --- | --- |
| 规划 + 执行 | `deepseek-v4-flash` | `agent.llm.executor` | 当前规划器复用执行器模型 |
| 标题生成 | `deepseek-v4-flash` | `agent.llm.planner` | 轻量调用 |
| 图片观察 | `agnes-2.0-flash` | `agent.llm.vision` | 只用于 view_image Post Hook |
| Embedding | `embedding-3` | `rag.embedding` | 智谱，维度 2048 |
| Query Rewrite | 当前回退主流程 DeepSeek 配置 | `rag.enhancement.rewrite` | 默认不是 Ollama |
| Rerank | 默认 DeepSeek `deepseek-v4-flash`，可切换 BGE | `rag.enhancement.rerank` | 独立模型配置与 provider |

注意：Query Rewrite 当前默认回退到主流程 DeepSeek 配置，api-url 优先 `RAG_REWRITE_URL`，回退 `DEEPSEEK_LLM_URL`。旧文档里写的 Ollama/qwen3 已不适用。Rerank 不继承主流程模型名，默认由 `RAG_RERANK_MODEL=deepseek-v4-flash` 控制。

## 4. 基础设施依赖

| 依赖 | 默认值 | 说明 |
| --- | --- | --- |
| MySQL | 库名 `sandbox_agent` | 业务元数据持久化 |
| Milvus | 默认关闭 | 向量数据库，`rag.milvus.enabled=false` 时由 Noop 降级 |
| 本地上传根目录 | `./uploads` | `agent.storage.local.base-path` |
| 用户上传目录 | `uploads/users/{userId}/uploads` | 用户级路径 |
| 用户知识库目录 | `uploads/users/{userId}/knowledge/{kbId}` | 用户级路径 |
| Sandbox 内部目录 | `/home/gem/uploads` 等 | 见 [[Sandbox 模块]] 目录结构 |
| BGE Rerank 服务 | `http://127.0.0.1:8002/rerank` | 仅 `RAG_RERANK_PROVIDER=bge` 时需要 |

## 5. 外部 API 依赖

| 服务 | 地址 | 环境变量 | 是否必需 |
| --- | --- | --- | --- |
| DeepSeek API | `https://api.deepseek.com` | `DEEPSEEK_LLM_URL` | 是 |
| DeepSeek API Key | - | `DEEPSEEK_API_KEY` | 是 |
| Agnes 视觉 API | `https://apihub.agnes-ai.com/v1` | `AGNES_LLM_URL` | 图片观察时是 |
| Agnes API Key | - | `AGNES_API_KEY` | 图片观察时是 |
| 智谱 Embedding | `https://open.bigmodel.cn/api/paas/v4/embeddings` | `RAG_EMBEDDING_URL` | RAG 检索时是 |
| 智谱 Embedding Key | - | `RAG_EMBEDDING_KEY` | RAG 检索时是 |
| OpenSandbox 平台 | `localhost:8080` | `SANDBOX_DOMAIN` | 是 |
| 百度搜索 OCR | - | `BAIDU_OCR_AK/SK` | OCR 功能时是 |

## 6. 面试表达重点

这个项目不是单纯调用大模型接口，而是一个多层工程系统：模型编排、工具协议、沙箱隔离、RAG、MCP 动态扩展、前端实时交互和文件预览都需要协同。准备面试时可从 [[项目面试学习路线]] 进入。
