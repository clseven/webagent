# 技术栈与外部依赖

## 后端技术栈

- Java 17。
- Spring Boot 3.2.5。
- Spring Web：HTTP API 和静态资源。
- Spring WebFlux：模型、RAG、MCP 等 HTTP 客户端调用。
- Spring WebSocket：沙箱视图同源代理转发 code-server、noVNC、Web Terminal 的 Upgrade 请求。
- Spring Data JPA + MySQL：业务元数据持久化。
- Lombok：简化实体和配置类。
- Apache Tika 2.9.1：知识库文档解析。
- Milvus Java SDK 2.4.1：可选向量数据库。
- MCP Java SDK 2.0.0：MCP Client、stdio/HTTP transport 和 JSON-RPC 类型。
- Jsoup：WebSearchTool 解析搜索结果。
- Alibaba OpenSandbox SDK + AIO REST：创建和操作沙箱。

## 前端技术栈

- 静态 Vue 3 + Vue Router，直接放在 `src/main/resources/static`。
- 无独立 `package.json` 和 Node 构建步骤。
- `fetch + ReadableStream` 处理 SSE，因为需要携带 Authorization Header。
- Highlight.js CDN 用于代码高亮。
- 文件预览由 `FilePreviewer.js` 统一承载，图片、PDF、文本、Markdown、CSV、Office 转 PDF 共用同一个入口。

## 模型和智能体依赖

- 默认 planner/executor：DeepSeek `deepseek-v4-flash`。
- 标题生成：`plannerLlm`，默认也可使用 DeepSeek 配置。
- 视觉观察：Agnes `agnes-2.0-flash`，通过 `visionLlm` 接入。
- Embedding：智谱 `embedding-3`，维度 2048。
- Query Rewrite：Ollama `qwen3:1.7b`。
- Rerank：BGE rerank 服务，默认 `/rerank`。

## 基础设施依赖

- MySQL 默认库名：`sandbox_agent`。
- 本地上传根目录默认：`./uploads`。
- 用户上传目录：`uploads/users/{userId}/uploads`。
- 用户知识库目录：`uploads/users/{userId}/knowledge/{kbId}`。
- Sandbox 内部目录：`/home/gem/uploads`、`/home/gem/workspace`、`/home/gem/output`、`/home/gem/skills`、`/home/gem/tools`、`/home/gem/knowledge`。

## 面试表达重点

这个项目不是单纯调用大模型接口，而是一个多层工程系统：模型编排、工具协议、沙箱隔离、RAG、MCP 动态扩展、前端实时交互和文件预览都需要协同。准备面试时可从 [[项目面试学习路线]] 进入。
