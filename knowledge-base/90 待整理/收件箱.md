# 收件箱

这里用于临时放置尚未确认归属的发现。

整理规则：

1. 能从代码、配置、测试或契约确认的内容，移动到对应模块或流程页并标记 `verified`。
2. 需要运行验证或产品判断的内容，移动到 [[待确认问题]] 或 [[当前已知风险]]。
3. 不要让收件箱成为长期文档；每次知识库维护都应清理。
