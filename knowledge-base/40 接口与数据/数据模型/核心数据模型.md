# 核心数据模型

## 用户和会话

- `User`：登录用户。
- `ConversationSessionEntity`：会话，保存 userId、title、appId、enabledSkillIds 等。
- `ChatMessage`：对话消息，支持 role、content、reasoning、events 和 vision content parts。

会话属于用户，但 Sandbox 资源按用户复用，不随会话删除销毁。

## Agent 应用

- Agent 应用保存用户自定义的应用配置。
- 应用可关联多个知识库和多个 Skill。
- 会话选择应用后，应用配置影响 AgentTurnContext。

## Sandbox

- `UserSandboxEntity`：保存 userId、sandboxId、aioEndpoint、deleted 等状态。
- `AioSandboxStore`：内存 endpoint 缓存，不是数据库实体。

当前用户级沙箱绑定是 Sandbox 模块的核心状态。`sessionSandboxMap` 等旧会话级内存结构不再作为长期资源归属来源。

## RAG

- 知识库实体：归属 userId。
- 文档实体：归属 userId 和 kbId，保存原始文件信息。
- 切片实体：保存文档切片内容、索引、偏移和元数据。
- 向量：存储在 Milvus，关闭 Milvus 时由 Noop 向量服务跳过。

## TodoState

运行时 Agent TodoState 不是数据库持久化状态，而是会话内内存状态：

- `AgentTodoState`：一次会话当前轮 todo 快照。
- `AgentTodoItem`：id、title、status、successSignals、evidence、blocker、reason、batchIds。
- `AgentTodoStatus`：pending、in_progress、completed、blocked、cancelled。
- `AgentTodoUpdateResult`：更新结果、警告和摘要。

TodoState 主要服务最终回答门禁和多步任务闭环。

## MCP

MCP 配置和运行态主要通过服务对象管理：

- `McpServerConfig`：server 配置。
- `McpClientKey`：系统级或用户级连接 key。
- `McpServerView`：给前端和工具返回的 server 视图。
- `RealMcpTool`：MCP 工具的 Tool 适配层。
