# Agent 时间感知与过程状态文案设计

## 目标

1. 对话执行期间和完成后使用同一套完成式状态文案，不再显示“思考中”“搜索中”“0 进行中”或工具数量。
2. 让规划器、执行器、社交轮和子代理共享同一份单轮时间快照，并提供按需查询精确时间的 `current_time` 工具。

## 前端交互

- 正常执行期间，过程总标题固定显示“已运行”；执行完成后历史消息仍显示“已运行”。
- 是否仍在执行由展开状态、活动圆点和加载动画表达，文案不切换时态。
- 搜索工具无论正在调用还是已经返回，都显示“已搜索网页（查询词）”；工具返回后可继续展示结果 URL。
- 思考/推理轮的摘要统一显示“已运行”。
- 网络断线属于异常状态，继续显示“连接中断，正在同步结果…”，避免把故障伪装成正常运行。
- 完成后沿用现有历史消息折叠行为。

## 时间上下文

- 每次进入 `AgentTurnContextService.prepare` 时生成一次不可变时间快照，默认时区为 `Asia/Shanghai`，可通过 `agent.time-zone` 配置覆盖。
- 快照包含本地日期、带偏移的本地时间、IANA 时区、星期和 UTC 时间，并明确相对日期按该时区解释。
- 同一轮快照进入规划器、普通执行器和社交轮；父 `ReactAgent` fork 子代理时继续传递同一快照。
- `current_time` 工具用于长任务、精确到分钟或跨时区查询；它每次调用都读取新的时间，并允许传入 IANA 时区。
- 时间上下文放在 `ReactPromptAssembler` 的动态边界之后，不改变稳定提示词前缀。
- “知道当前日期”和“知道最新事实”严格区分；最新政策、版本、新闻等仍需搜索验证。

## 测试

- Node 纯函数测试覆盖“已运行”“已搜索网页（查询词）”、历史 `toolResult` 完成识别和无工具计数。
- JUnit 使用固定 `Clock` 验证时间格式、时区转换、非法时区错误、动态边界位置和社交轮注入。
- Maven 主代码编译、定向 JUnit、现有 Node 测试和静态资源缓存版本共同作为回归验证。
