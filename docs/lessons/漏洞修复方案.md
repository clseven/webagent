## 修复方案

### 1. Django 重定向漏洞 (CVE-2018-14574) — 中危

5601 通常是 Kibana 端口，先确认实际服务：

**确认为 Kibana** → 扫描器误报，加固 nginx 反代即可：

```nginx
location / {
    auth_basic "Access";
    auth_basic_user_file /etc/nginx/.htpasswd;
    proxy_pass http://127.0.0.1:5601/;
}
```

**确认为 Django** → 升级版本：

```bash
pip install --upgrade "Django>=4.2"
```

### 2. Swagger API 未授权访问 — 中危

**方案一：生产节点禁用 Swagger（最简单）**

```yaml
# application-prod.yml
springdoc:
  api-docs:
    enabled: false
  swagger-ui:
    enabled: false
```

**方案二：Spring Security / Shiro 认证授权**

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {
    @Bean
    public SecurityFilterChain swaggerFilterChain(HttpSecurity http) throws Exception {
        http
            .securityMatcher("/swagger-ui/**", "/swagger-resources/**",
                             "/v2/api-docs/**", "/v3/api-docs/**",
                             "/swagger-ui.html", "/webjars/**", "/doc.html")
            .authorizeHttpRequests(auth -> auth.anyRequest().authenticated())
            .httpBasic(Customizer.withDefaults());
        return http.build();
    }
}
```

```yaml
spring:
  security:
    user:
      name: swagger_admin
      password: <强密码>
```

**方案三：nginx / Filter 访问控制**

```nginx
location ~* ^/(swagger-ui|swagger-resources|v2/api-docs|v3/api-docs|webjars|doc\.html) {
    allow <办公网IP段>;
    deny all;
    # 或加 Basic Auth
    auth_basic "Swagger";
    auth_basic_user_file /etc/nginx/.htpasswd;
}
```

### 3–5. SSL/TLS 问题（弱DH密钥 + LogJam + 弱算法）— 一处修复覆盖全部

```bash
openssl dhparam -out /etc/nginx/dhparams.pem 2048
```

```nginx
server {
    listen 9300 ssl;
    ssl_dhparam /etc/nginx/dhparams.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_prefer_server_ciphers on;
    ssl_ciphers 'ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256:ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384:ECDHE-ECDSA-CHACHA20-POLY1305:ECDHE-RSA-CHACHA20-POLY1305:DHE-RSA-AES128-GCM-SHA256:DHE-RSA-AES256-GCM-SHA384';
}
```

```bash
nginx -t && nginx -s reload
```

### 6. HTTP Banner 信息泄露 — 低危

> 报告说明：属于信息获取类漏洞，若 banner 未包含敏感信息，可**不进行修复**。

如需修复：

```nginx
# nginx.conf
http {
    server_tokens off;
}
```

```yaml
# Spring Boot application.yml
server:
  server-header: false
```

### 7. MySQL 版本泄露 — 低危

MySQL 5.7 已于 2023年10月 EOL，建议升级到 8.0：

```ini
# my.cnf — MySQL 8.0 隐藏版本信息
version_compile_os = none
```

短期先限制远程访问：

```ini
bind-address = 127.0.0.1
```

---

### 8. 未正确返回 404 — 低危

> 报告说明：此漏洞为对页面反应的判断，**可以不修复**。

如需修复：

```nginx
server {
    error_page 404 /404.html;
    location = /404.html { internal; }
    location / { try_files $uri $uri/ =404; }
}
```
